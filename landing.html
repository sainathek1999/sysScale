How it works

The 5-step framework

Every system follows the same methodology. Build it into muscle memory.

1

Clarify scope

Pin the 5 key decisions before estimating anything: where limiting happens, granularity, storage backend, consistency model, failure behavior. Interviewers notice when you ask first.

2

Traffic estimation

Calculate average RPS from DAU and usage patterns. Every system has a characteristic peak multiplier — rate limiters ×5, notifications ×10, video conferencing O(N²).

3

Storage estimation

Bytes per entity × entity count. Know the difference: Redis key at 20B, Cassandra row at 1KB, S3 recording at gigabytes. Hot vs cold changes the architecture.

4

Bandwidth estimation

Peak RPS × payload × 8. Separate inbound from outbound. Determine whether the bottleneck is compute, memory, or network — each implies a different scaling strategy.

5

Scale and nodes

Total load ÷ single-node capacity. Add replication. Name the bottleneck explicitly. SysScale flags it automatically — training you to recognize the pattern before an interviewer points it out.
